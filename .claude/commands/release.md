---
description: Cut a new pyloros release (bump Cargo version, PR, tag, push)
---

Cut a new release of pyloros. Arguments: `$ARGUMENTS` — optional explicit version like `0.5.0`. If omitted, propose one based on commits since the last tag.

## Procedure

1. **Inspect state**
   - Current version: `grep '^version' Cargo.toml | head -1`
   - Last tag: `git tag --sort=-v:refname | grep '^v' | head -1`
   - Changes since last tag: `git log <last-tag>..main --oneline`
   - Decide next version (semver; breaking → major, features → minor, fixes → patch). Confirm with the user if not explicitly given.

2. **Create a worktree** (required — never edit main directly, see CLAUDE.md):
   ```bash
   git branch claude/release-vX.Y.Z main
   git worktree add ../pyloros-worktrees/release-vX.Y.Z claude/release-vX.Y.Z
   cd ../pyloros-worktrees/release-vX.Y.Z && lefthook install
   ```

3. **Bump version**
   - Edit `Cargo.toml`: `version = "X.Y.Z"`.
   - Update lockfile: `cargo update -p pyloros --precise X.Y.Z`.
   - Do NOT edit any CHANGELOG — release notes are auto-generated by the GitHub Release workflow (`generate_release_notes: true` in `.github/workflows/release.yml`).

4. **Sanity-check build** (fast): `cargo build` and `cargo test --lib`. Skip the full integration suite — CI reruns everything on the tag.

5. **Commit and PR**
   ```bash
   git add Cargo.toml Cargo.lock
   git commit -m "release: vX.Y.Z"
   git push -u origin claude/release-vX.Y.Z
   gh pr create --title "release: vX.Y.Z" --body "Bump version to X.Y.Z. Release notes auto-generated on tag push."
   ```
   Wait for CI to go green, then merge (squash or merge-commit — project default).

6. **Tag and push** (AFTER the PR is merged to main):
   ```bash
   git checkout main && git pull
   git tag vX.Y.Z
   git push origin vX.Y.Z
   ```
   The `release.yml` workflow validates `tag == Cargo.toml version`, builds the musl binary, creates the GitHub Release with auto-generated notes, and pushes `ghcr.io/pyloros-org/pyloros:vX.Y.Z` + `:latest`.

7. **Verify**: `gh release view vX.Y.Z` and confirm the tarball + SHA256SUMS + docker image are present.

8. **Cleanup**:
   ```bash
   git worktree remove ../pyloros-worktrees/release-vX.Y.Z
   git branch -d claude/release-vX.Y.Z
   ```

## Notes

- The release workflow auto-generates notes from merged PRs — keep PR titles clean.
- The `latest` tag and `:edge` docker image are updated on every main push by a separate job; no manual action needed.
- If the tag/Cargo.toml mismatch validation fails, delete the tag (`git push origin :refs/tags/vX.Y.Z`), fix, and retag.
